# CHANGELOG

## [1.2.0] — 2026-03-20

### Добавлено
- Кнопка `🚫 Заблокировать` в сообщении заявки в группе
- Обработчик `admin_block:{app_id}`: проверяет права, блокирует пользователя, отклоняет заявку, уведомляет пользователя
- Колонка `is_blocked` в таблице `users` (с автомиграцией для существующих БД)
- Функции в `database.py`: `block_user`, `unblock_user`, `is_user_blocked`, `get_all_users_stats`
- Проверка блокировки при нажатии «Подать заявку» (`start.py`)
- **`clanpanel.py`** — терминальная панель управления: список пользователей, блокировка/разблокировка по TG ID, просмотр заявок с фильтром по статусу
- Автомиграция БД при запуске `clanpanel.py` (добавляет `is_blocked` в старые БД без потери данных)

## [1.0.0] — 2026-03-20

### Первый релиз

#### Функциональность
- FSM-сценарий подачи заявки: игровой ID, ник, ранги (Competitive / Allies / Duels), часы, KD
- Автоматическое отклонение при KD < 1.0 / > 10 или часах < 50 (статус `rejected_auto`)
- Подтверждение заявки с inline-меню редактирования любого поля
- Анти-спам: не чаще 1 заявки в 10 минут
- Блокировка повторной подачи при наличии активной заявки (`pending`)
- Отправка заявки в Telegram-группу с кнопками `✅ Принять` / `❌ Отклонить`
- Проверка прав через `getChatMember` перед каждым действием
- При принятии — генерация одноразовой invite-ссылки (24 ч, 1 использование)
- Поддержка форум-групп с темами (`TOPIC_ID`)
- Хранение данных в SQLite через `aiosqlite`
- Загрузка конфигурации из `.env` (`BOT_TOKEN`, `GROUP_ID`, `TOPIC_ID`)

#### Стек
- Python 3.11+, aiogram 3.x, aiosqlite, python-dotenv, rich
